# README Visual Punch List

Prioritized ideas for assets that would make the landing page feel more polished and easier to scan.

## Priority 1

- Hero screenshot: Godot editor with the Godot AI dock visible and an MCP client prompt alongside it.
- Primary demo GIF: prompt to create a node, then show the scene tree update in the editor.
- Dock panel screenshot: call out connection state, Configure buttons, and the log area.

## Priority 2

- Architecture diagram: replace the ASCII flow in `README.md` with a simple branded SVG.
- Before and after pair: empty scene, then the same scene after an AI-assisted edit.
- Resource demo screenshot: a client reading `godot://scene/current` or `godot://project/settings`.

## Priority 3

- Multi-session GIF: two Godot editors connected, then active-session switching.
- Test runner GIF: `run_tests` from an MCP client, followed by structured results.
- Filesystem search screenshot: example query and paginated results for `filesystem_search`.

## Notes

- Keep visuals product-oriented, not just code-oriented.
- Favor short GIFs that show a complete loop in under 10 seconds.
- Use consistent window chrome, font sizing, and crop widths so multiple images feel like one system.
- Avoid stuffing the README with too many badges or screenshots before the first successful-use-path is clear.
